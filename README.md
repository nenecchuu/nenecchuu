### ⚙️ Engineering principles

- **Ship, then iterate.** A working v0 in a week beats a perfect v1 never.
- **Own the full slice.** From schema to deploy to monitoring — not just "the frontend part."
- **Automate the repetitive.** Cron jobs, scripts, GH Actions. If I do it twice, the third time is code.
- **Read the codebase before writing it.** Most "hard problems" are solved by someone in the same repo.
- **Tests where they pay rent.** Unit tests for logic, integration for boundaries, E2E for the critical path.
